# АІС «ZLAGODA» — керування збутом товарів у міні-супермаркеті

Автоматизована інформаційна система для регулювання роботи продуктового
міні-супермаркету «ZLAGODA». Система обробляє дані процесу збуту товарів:
облік товарів і їх кількості, ціни продажу, історія чеків, категорії товарів,
працівники та постійні клієнти з картками.

Система орієнтована на непрофесійного користувача (не програміста) і забезпечує
дружній інтерфейс для касирів та менеджерів.

---

## Технології

- **FastAPI** — вебзастосунок (Python)
- **SQLite** — база даних (`zlagoda.db`)
- **Чисті SQL-запити** — без ORM; уся робота з даними виконується параметризованими SQL-запитами
- **Jinja2** — шаблони інтерфейсу користувача
- **JWT + passlib (bcrypt)** — аутентифікація та хешування паролів

---

## Користувачі та права доступу

Система розрахована на дві категорії користувачів.

### Менеджер
Контролює всі процеси в супермаркеті. Має право:
- додавати, редагувати, видаляти дані працівників, клієнтів, категорій, товарів, товарів у магазині;
- видаляти чеки;
- формувати та друкувати звіти;
- виконувати всі пошукові й аналітичні запити (списки, суми продажів за період, кількість проданого товару тощо).

### Касир
Відповідає за продаж товарів. Має право:
- здійснювати продаж — створювати чеки (це право належить **лише** касиру);
- додавати та редагувати інформацію про постійних клієнтів;
- переглядати товари, товари в магазині, клієнтів, виконувати пошук;
- переглядати власні чеки за день / за період;
- переглядати інформацію про себе.

Право на створення чеків має тільки касир; решта операцій введення/видалення —
лише менеджер. Оновлення картки клієнта доступне обом ролям.

## Структура бази даних

Сім таблиць (6 сутностей + звʼязок «Продаж»):

| Таблиця | Призначення |
|---|---|
| `Category` | Категорії товарів |
| `Product` | Товари (назва, виробник, характеристики, категорія) |
| `Store_Product` | Товари у магазині (UPC, ціна, кількість, звичайний/акційний) |
| `Employee` | Працівники (менеджери та касири) + хеш пароля |
| `Customer_Card` | Картки постійних клієнтів зі знижкою |
| `Check` | Чеки (номер, дата, загальна сума, ПДВ) |
| `Sale` | Продаж — товари та їх кількість і ціна в кожному чеку |

### Бізнес-правила
- **ПДВ** = загальна сума × 0.2 (входить у ціну продажу).
- **Акційна ціна** = звичайна ціна × 0.8 (знижка 20%).
- **Переоцінка**: при надходженні нової партії з новою ціною всі одиниці товару отримують нову ціну продажу; у чеках зберігається ціна на момент продажу.
- Кожному товару може відповідати не більше двох записів у магазині: звичайний та акційний.
- Чеки зберігаються протягом 3 років.

### Обмеження цілісності
- Вік працівника — не менше 18 років.
- Номер телефону — не довше 13 символів (включно з «+»).
- Ціна, кількість, зарплата, відсоток не можуть бути відʼємними.

## Установлення та запуск

```bash
# 1. Клонувати репозиторій
git clone https://github.com/USERNAME/zlagoda-ais.git
cd zlagoda-ais

# 2. Створити та активувати віртуальне середовище
python3 -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

# 3. Встановити залежності
pip install -r requirements.txt

# 4. Створити базу даних з тестовими даними
python init_db.py

# 5. Запустити сервер
uvicorn main:app --reload
```

Після запуску відкрити у браузері:
- `http://127.0.0.1:8000/login` — сторінка входу
- `http://127.0.0.1:8000/docs` — Swagger (тестування API)

### Тестові облікові записи
| ID | Пароль | Роль |
|---|---|---|
| `EMP001` | `admin` | Менеджер |
| `EMP002` | `1234` | Касир |

## Звіти

Менеджер може формувати звіти про працівників, клієнтів, категорії, товари,
товари у магазині та чеки. Звіти відповідають вимогам:
- кнопка звіту відкриває **попередній перегляд**, а не одразу друк;
- наявні верхній і нижній колонтитули;
- таблиця займає всю ширину сторінки;
- адреса сторінки (URL) у звіті не друкується.

## Структура проєкту

```
zlagoda-ais/
├── main.py             # точка входу FastAPI, підключення роутерів
├── database.py         # підключення до SQLite
├── init_db.py          # створення схеми та тестових даних
├── security.py         # хешування паролів, JWT
├── ui_routes.py        # маршрути сторінок інтерфейсу
├── requirements.txt    # залежності
├── templates/          # HTML-шаблони (Jinja2)
│   ├── base.html
│   ├── login.html
│   └── ...
└── static/             # CSS, іконки, зображення
    └── css/style.css
```

База даних `zlagoda.db` та `venv/` не зберігаються в репозиторії —
БД генерується локально через `python init_db.py`.

---

## Безпека

- Паролі **ніколи** не зберігаються у відкритому вигляді — лише bcrypt-хеш.
- Розрізняються ідентифікація та аутентифікація користувача.
- Доступ до операцій обмежено відповідно до ролі (менеджер / касир).
